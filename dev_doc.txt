6.  Automated  AI  Code  Review:  Use  git-lrc  and  review  your  
code
 
(20
 
minutes)
 
M
 
During  development,  students  will  also  learn  the  importance  of  verifying  AI-generated  code .  
Although  AI  tools  can  generate  code  quickly,  they  can  sometimes  introduce:  
●  bugs  ●  security  issues  ●  inefficient  logic  ●  unnecessary  complexity  
To  address  this,  the  workshop  introduces  git-lrc ,  an  AI-powered  code  review  system.  
Students  will  run  a  review  on  their  generated  code  before  committing  it.  
They  will  see:  
●  inline  review  comments  ●  detected  issues  ●  Suggestions  for  improvement  
This  reinforces  an  important  principle:  
AI  should  assist  development,  but  code  should  always  be  reviewed  and  
verified.
 
Students  will  then  commit  their  code  using  Git.  
Example  workflow:  
Install;  https://github.com/HexmosTech/git-lrc?tab=readme-ov-file#install 
git  add  .  git  lrc  review 
●  A  review  UI  will  open  ●  git-lrc  reviews  your  code  automatically  ●  If  any  issues  are  flagged,  copy  the  suggestions  and  fix  them  using  your  AI  assistant  
(e.g.,
 
Copilot)
 ●  Stage  the  changes  again:  
git  add  .  git  lrc  review 
●  The  code  will  be  reviewed  again  ●  Once  critical  issues  are  resolved,  proceed  with  the  Commit  &  Push  button  in  the  UI  
https://github.com/HexmosTech/git-lrc
